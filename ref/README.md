# Reference Files

Development-only reference files from upstream projects. These document the patterns and conventions that Void UI is based on. NOT shipped in any package.

## Contents

- `palette/` — Starting color palette and design token values
- `tokens/` — Token tier structure (primitives → semantic)
- `components/` — Lit component authoring patterns (Light DOM base class, component examples)
- `adapters/` — Framework adapter patterns (React wrapper, SolidJS composables)
- `build/` — Package.json and Vite build configuration patterns
- `ci/` — Release workflow patterns (per-package semver tagging)
- `federation/` — Module Federation integration patterns
